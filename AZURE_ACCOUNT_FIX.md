# Решение проблемы с входом в Azure

## Проблема

При попытке войти в Azure Portal вы видите ошибку:
> "Выделенная учетная запись пользователя не существует в клиенте "Microsoft Services" и не может обращаться к приложению..."

Это означает, что ваш email (mail.ru, yandex.ru и т.д.) не является Microsoft аккаунтом и не может использоваться для Azure.

## Решение 1: Создать новый Microsoft аккаунт (Рекомендуется)

⚠️ **Важно:** Аккаунты `live.com`, `hotmail.com` и некоторые старые Microsoft аккаунты могут не работать с Azure Portal. Лучше создать новый аккаунт с доменом `@outlook.com`.

### Шаг 1: Создание нового Microsoft аккаунта

1. Перейдите на [account.microsoft.com](https://account.microsoft.com/)

2. **Выйдите из текущего аккаунта** (если вы вошли)

3. Нажмите **"Создать учетную запись Microsoft"** или **"Sign up"**

4. Выберите вариант:

   **Рекомендуется: Создать новый email с доменом @outlook.com**
   - Выберите **"Получить новый адрес электронной почты"**
   - Выберите домен: **`@outlook.com`** (рекомендуется, так как он точно работает с Azure)
   - Придумайте имя пользователя (например: `findorigin2024`)
   - Создайте надежный пароль
   - Заполните остальные данные (имя, дата рождения, страна)
   - Подтвердите, что вы не робот
   - Завершите регистрацию

   ⚠️ **Не используйте:** `live.com` или старые домены - они могут не работать с Azure Portal

4. Завершите регистрацию

### Шаг 2: Вход в Azure

1. Перейдите на [Azure Portal](https://portal.azure.com/)

2. Войдите используя ваш новый Microsoft аккаунт

3. Создайте бесплатный Azure аккаунт:
   - Нажмите "Start free" или "Начать бесплатно"
   - Заполните форму (имя, телефон, страна)
   - Добавьте способ оплаты (кредитная карта)
     - ⚠️ **Важно:** С бесплатного тарифа деньги не списываются
     - Вы получите $200 кредитов на 30 дней
     - После этого бесплатные сервисы остаются бесплатными

## Решение 2: Использовать альтернативные API

Если создание Microsoft аккаунта проблематично, используйте другие варианты:

### Вариант A: Google Custom Search API

1. Перейдите на [Google Cloud Console](https://console.cloud.google.com/)
2. Создайте проект
3. Включите Custom Search API
4. Создайте API ключ
5. Создайте поисковую систему на [programmablesearchengine.google.com](https://programmablesearchengine.google.com/)

**Инструкция:** См. файл `GOOGLE_API_SETUP.md`

**Лимит:** 100 запросов в день бесплатно

### Вариант B: Прямой парсинг (для тестирования)

Можно временно использовать прямой парсинг поисковых систем для тестирования, но это:
- ⚠️ Может нарушать условия использования
- ⚠️ Менее надежно
- ⚠️ Требует больше кода

## Решение 3: Использовать VPN (на свой риск)

Если у вас есть VPN, можно попробовать:
1. Включить VPN
2. Попробовать войти в Azure снова
3. ⚠️ **Внимание:** Использование VPN может нарушать условия использования сервисов

## Рекомендация

**Лучший вариант:** Создайте Microsoft аккаунт с email `@outlook.com` или `@hotmail.com`. Это:
- ✅ Бесплатно
- ✅ Занимает 5 минут
- ✅ Работает надежно
- ✅ Дает доступ ко всем Microsoft сервисам

## Быстрая инструкция по созданию Microsoft аккаунта

1. Откройте: https://account.microsoft.com/
2. Нажмите "Создать учетную запись"
3. Выберите "Получить новый адрес электронной почты"
4. Выберите `@outlook.com`
5. Придумайте имя (например: `findorigin2024@outlook.com`)
6. Создайте пароль
7. Заполните остальные данные
8. Подтвердите email (если потребуется)
9. Готово! Теперь используйте этот аккаунт для Azure

## После создания Microsoft аккаунта

1. Войдите в [Azure Portal](https://portal.azure.com/) используя новый аккаунт
2. Создайте ресурс "Bing Search v7"
3. Получите API ключ
4. Добавьте в `.env.local`:
   ```env
   BING_SEARCH_API_KEY=ваш_ключ
   ```
